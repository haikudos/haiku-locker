SIGNER KEY GENERATION — OFFLINE ONLYSENSITIVE

Signer key helper

Generates the secp256k1 key the signer process uses. This is a distinct key from your funded manager-admin wallet. It must live on the signer node, so it is inherently more exposed — keep it isolated from the Ledger-backed wallet, and generate it where the node will run.

Read before using. A browser is not the ideal place to make a production private key. Preferred path is the node host, offline:
  1. On the signer host, generate a secp256k1 key with your standard tooling (e.g. openssl rand -hex 32 for the raw key, or the key format your stacks-signer build documents), then place it in the signer config per the signer guide.
  2. Only use this page if you need a helper and can run it fully offline / air-gapped. Turn off Wi-Fi, generate, transfer via trusted channel, wipe the machine's clipboard and this tab.
Never paste this private key into a website, the other dApps in this set, chat, or email. The other dApps only ever need the public key.
